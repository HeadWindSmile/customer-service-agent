# AGENTS.md

## 项目定位

这是一个用于 Agent 开发岗位面试的企业级 AI 客服问答系统 Demo。

项目目标不是普通 ChatBot，而是在原有业务系统旁边新增 Python/FastAPI AI 服务层，模拟“业务微服务 + AI 服务层（LLM + Agent）”融合架构。

## 固定开发原则

1. 必须基于当前已有代码增量开发，不要推翻重写整个项目。
2. 不要大规模重命名目录或文件，除非任务明确要求。
3. 不要把业务逻辑写到 main.py 或 api 层。
4. api 层只负责请求接收、参数校验、调用 agent、返回响应。
5. customer_agent.py 负责主编排。
6. router.py 负责根据 intent 分发。
7. tools 层模拟或调用业务系统能力。
8. rag 层负责知识库加载、切分、检索、sources 返回。
9. memory 层负责会话记忆。
10. safety 层负责输入输出安全。
11. observability 层负责 trace、日志、耗时、tool_calls。
12. 每个新增模块必须有必要注释，注释重点解释“为什么这样设计”，方便面试讲解。
13. 不要生成前端页面。
14. 不要写伪代码，代码必须真实可运行。
15. 外部依赖必须提供 mock/fallback 模式，保证本地最小版本能启动。

## 每阶段完成标准

每个阶段完成后必须：

1. 更新 README 的当前阶段能力和启动方式。
2. 提供至少 3 个可运行 curl 示例。
3. 补充或更新 pytest 测试。
4. 如果新增环境变量，必须同步更新 .env.example。
5. 输出修改文件清单。
6. 输出启动命令。
7. 输出测试命令。
8. 输出核心接口验证方式。

## 本地验收命令

优先使用以下命令验证：

```bash
pytest
uvicorn app.main:app --reload
```

如果使用 docker compose，则补充：

```bash
docker compose up -d
docker compose ps
```

## 禁止行为

1. 禁止为了让测试通过而删除重要测试。
2. 禁止把真实业务能力写死在 API 层。
3. 禁止把 mock 写成无法替换的结构。
4. 禁止跳阶段开发。
5. 禁止未经说明引入大型复杂依赖。
6. 禁止声称 Demo 已支持生产级高并发。
7. 禁止生成与当前阶段无关的大量代码。
